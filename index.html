EverFit

An accessible fitness and physical therapy app designed to support elderly users through intuitive design and motivational tools.

Project Overview

EverFit addresses the growing need for accessible fitness tools for aging populations. Many elderly individuals struggle with mobility and motivation, especially when using apps not built with their needs in mind. Our app bridges this gap with simple navigation, guided workouts, and a friendly assistant.

  • Goals: Improve mobility, provide motivation, and reduce injury risk.
  • Target Users: Elderly individuals with limited tech experience.
  • Features: Exercise tracking, assistant chat, progress visualizations, and more.

Design Process

We followed a human-centered design process, including:

  • User interviews and surveys to understand elderly needs and pain points.
  • Personas representing different levels of tech familiarity and mobility.
  • User journey maps and storyboards outlining common app interactions.
  • Multiple rounds of design iterations based on user feedback and heuristic evaluations.

Prototyping

We created a high-fidelity prototype in Figma to demonstrate all core user flows, from tracking workouts to receiving assistance and viewing progress.

These screens were reviewed by peers and older adults, and their feedback helped refine text size, layout simplicity, and feature prioritization.

Heuristic Evaluation

  • Visibility of System Status: Progress bars and simple visual feedback improve user confidence.
  • Recognition Over Recall: Large icons and persistent navigation elements support memory constraints.
  • Error Prevention: Confirmations for risky actions (like resetting data) were added based on feedback.
  • Suggested Improvements: Increased contrast, simplified menu labels, and larger tap targets.

Reflections & Conclusion

EverFit reinforced the importance of empathy in the design process. By centering the needs of elderly users, we built something simple, functional, and motivating. Key lessons included the value of early user feedback, the power of visual hierarchy, and how thoughtful details (like language and layout) can make technology feel more human.

We're proud of how this prototype responds to real-world accessibility needs and hope to see similar principles adopted more widely.
